# OpenBodhi Knowledge Ontology

The vault is a typed graph. Every node has a type. Every edge has a direction and meaning.
This is not a flat list of notes. It is a model of how ideas relate to each other over time.

---

## Node Types

Six types. Chosen for how ideas actually move through a mind, not how productivity tools categorize tasks.

| Type | Color | Description |
|------|-------|-------------|
| Idea | `#d4941a` amber | Raw insight. Not yet verified. The thought as it arrived. |
| Pattern | `#cca329` gold | A recurring theme. Appears across multiple Ideas over time. |
| Practice | `#5a8a75` sage | Something you do intentionally. A habit, ritual, or discipline. |
| Decision | `#508cb4` blue | A choice made, with context. Captures what you chose and why. |
| Synthesis | `#8c64aa` purple | AI-discovered connection. Generated by Surveyor or Distiller. |
| Integration | `#558c55` green | An insight you have embodied. Moved from knowing to being. |

### Type semantics

**Idea** is the entry point. Most captures start here. The Curator assigns this type when
the content is exploratory, provisional, or fragmentary. Energy level captures urgency.

**Pattern** is emergent. The Distiller surfaces these from repeated Ideas. A Pattern cannot
be created directly — it is discovered. The edge type SURFACES_FROM connects the Pattern
back to the Ideas that revealed it.

**Practice** is action-oriented. When an Idea or Pattern crystallizes into something you
actually do, it becomes a Practice. Practices tend to have low variance in energy level
because they are stable behaviors, not impulses.

**Decision** has a timestamp and context. It answers: what did I choose, and what led me there?
Decisions are connected to the Ideas and Patterns that informed them via INFORMS edges.

**Synthesis** is the Surveyor's output. A bridge node connecting ideas that cluster near each
other in vector space but have no explicit edge. Claude labels these with a one-line synthesis.
The human may promote a Synthesis to Pattern or Integration.

**Integration** is the rarest type. It marks the transition from intellectual understanding
to lived embodiment. Only a human can assign Integration type — not the system.

## Relationship Types

Six edge types. Direction matters.

| Type | Direction | Description |
|------|-----------|-------------|
| LEADS_TO | A → B | A naturally develops into B over time |
| GENERATES | A → B | A directly produced or caused B |
| SURFACES_FROM | A → B | A became visible because of B (Synthesis → Ideas) |
| CONTRADICTS | A ↔ B | A and B are in tension. Bidirectional. |
| ASSOCIATED_WITH | A ↔ B | Related. No causal claim. Undirected. |
| INFORMS | A → B | A provides context for B without generating it |

### Edge semantics

**LEADS_TO** captures natural development. An Idea about meditation leading to a Practice
of morning breath work. Time is implied: A came before B, and A pointed toward B.

**GENERATES** is stronger. A Decision generates a set of actions. A Pattern generates a
new Idea when seen from a fresh angle. Causal, not just temporal.

**SURFACES_FROM** is the Surveyor's primary edge. A Synthesis node surfaces from the
cluster of Ideas that share vector proximity. It does not generate them — it is extracted from them.

**CONTRADICTS** is valuable. Productive tension in the vault is useful information.
Two Ideas that contradict each other create a Decision node: which do you actually believe?
The system does not resolve contradictions. It surfaces them.

**ASSOCIATED_WITH** is the weakest link. Used when two nodes relate but the nature of
the relationship is unclear. The Janitor reviews ASSOCIATED_WITH edges weekly and may
suggest promotion to a stronger type if a pattern is clear.

**INFORMS** is context without causation. Your reading on complexity theory informs your
approach to systems design, but did not generate the approach. Clean distinction.

## Required Fields Per Node

Every node in the vault must carry:

```json
{
  "id": "uuid-v4",
  "type": "Idea | Pattern | Practice | Decision | Synthesis | Integration",
  "content": "The thought, in the user's words",
  "energy_level": 3,
  "created_at": "2026-03-07T06:00:00Z",
  "source": "signal | whatsapp | manual",
  "tags": ["tag-one", "tag-two"]
}
```

| Field | Type | Notes |
|-------|------|-------|
| `id` | UUID v4 | Generated on creation. Immutable. |
| `type` | enum | One of 6 node types above |
| `content` | string | The raw thought. Preserved as captured, not edited by AI. |
| `energy_level` | integer 1-5 | User-assigned or inferred. 1 = low, 5 = high urgency/resonance. |
| `created_at` | ISO 8601 | Timestamp with timezone. |
| `source` | enum | Where the capture came from. |
| `tags` | string[] | AI-generated from content. Human may add or remove. |

### Optional fields

```json
{
  "updated_at": "2026-03-07T09:30:00Z",
  "promoted_from": "uuid-of-source-node",
  "cluster_id": "hdbscan-cluster-label",
  "embedding_model": "nomic-embed-text"
}
```

## Required Fields Per Edge

```json
{
  "id": "uuid-v4",
  "type": "LEADS_TO | GENERATES | SURFACES_FROM | CONTRADICTS | ASSOCIATED_WITH | INFORMS",
  "from": "source-node-uuid",
  "to": "target-node-uuid",
  "created_at": "2026-03-07T06:00:00Z",
  "created_by": "curator | distiller | surveyor | human",
  "confidence": 0.85
}
```

| Field | Type | Notes |
|-------|------|-------|
| `id` | UUID v4 | Generated on creation. |
| `type` | enum | One of 6 edge types above |
| `from` | UUID | Source node |
| `to` | UUID | Target node |
| `created_at` | ISO 8601 | When the edge was created |
| `created_by` | enum | Which worker or human created this edge |
| `confidence` | float 0-1 | For AI-generated edges only. Human edges default to 1.0. |
